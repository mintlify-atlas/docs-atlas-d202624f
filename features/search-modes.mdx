Scira offers 17 specialized search modes, each optimized for different types of research. The platform automatically selects the best mode based on your query, or you can manually choose one.

## General Research

<AccordionGroup>
  <Accordion title="Web" icon="globe" defaultOpen>
    **Purpose**: Search the entire web with AI-powered analysis
    
    The default mode for general research questions. Uses multiple search providers (Exa, Parallel, Tavily, Firecrawl) with intelligent deduplication and ranking.
    
    **When to use:**
    - General research questions
    - Current events and news
    - Product reviews and comparisons
    - How-to guides and tutorials
    
    **Example queries:**
    - "What are the latest developments in renewable energy?"
    - "Compare the top 5 project management tools"
    - "How does photosynthesis work?"
    
    **Features:**
    - Multi-query parallel search (5-10 queries)
    - Image search integration
    - Date/time context awareness
    - Source diversity and deduplication
  </Accordion>

  <Accordion title="Chat" icon="message">
    **Purpose**: Talk to the model directly, no search
    
    Pure conversational AI without web search. Useful for creative tasks, brainstorming, code generation, or questions that don't require real-time data.
    
    **When to use:**
    - Creative writing and brainstorming
    - Code generation and debugging
    - Mathematical calculations
    - General knowledge questions
    
    **Example queries:**
    - "Write a poem about artificial intelligence"
    - "Debug this Python function"
    - "Explain quantum entanglement simply"
    
    **Features:**
    - Direct model access
    - No search overhead
    - Faster responses
    - Best for well-defined problems
  </Accordion>

  <Accordion title="Extreme" icon="bolt">
    **Purpose**: Deep research with multiple sources and multi-step analysis
    
    Autonomous research agent that plans research strategy, executes searches, analyzes findings, and can run Python code for data analysis and visualizations.
    
    **When to use:**
    - Complex research requiring multiple perspectives
    - In-depth market analysis
    - Technical deep-dives
    - Research requiring data analysis or charts
    
    **Example queries:**
    - "Analyze the competitive landscape of AI chip manufacturers"
    - "Deep dive into the economics of sustainable aviation fuel"
    - "Compare battery technologies for electric vehicles with cost projections"
    
    **Features:**
    - Autonomous research planning (3-5 research phases)
    - Web and X search integration
    - Python code execution with chart generation
    - Multi-step reasoning and cross-validation
    - Source collection and content extraction
    - Configurable provider (Exa or Parallel)
    
    **Technical details:**
    - Uses Grok 4 for planning and execution
    - Sandboxed Python environment via Daytona
    - R2 artifact storage for charts
    - Step limit: 15-17 actions
  </Accordion>
</AccordionGroup>

## Academic & Technical

<AccordionGroup>
  <Accordion title="Academic" icon="graduation-cap">
    **Purpose**: Research papers, citations, and scholarly sources
    
    Specialized search for peer-reviewed papers, academic publications, and scholarly articles. Uses Exa and Firecrawl with category filtering.
    
    **When to use:**
    - Literature reviews
    - Scientific research
    - Citation discovery
    - Academic writing support
    
    **Example queries:**
    - "Recent papers on CRISPR gene editing in 2025"
    - "Literature review on transformer architectures in NLP"
    - "Academic research on climate change mitigation strategies"
    
    **Features:**
    - Peer-reviewed source prioritization
    - Citation extraction
    - Author and publication metadata
    - DOI and arXiv integration
  </Accordion>

  <Accordion title="Code" icon="code">
    **Purpose**: Get context about languages, frameworks, and programming topics
    
    Uses Exa's Context API to retrieve relevant programming documentation, examples, and best practices.
    
    **When to use:**
    - Learning new frameworks
    - API documentation lookup
    - Best practices research
    - Code examples and patterns
    
    **Example queries:**
    - "How to implement authentication in Next.js 15?"
    - "React Server Components best practices"
    - "Python asyncio patterns for concurrent API calls"
    
    **Features:**
    - Developer-focused content
    - Code snippet extraction
    - Documentation prioritization
    - Stack Overflow and GitHub integration
  </Accordion>

  <Accordion title="GitHub" icon="github">
    **Purpose**: Repositories, code, and developer discussions
    
    Search GitHub repositories with structured metadata extraction using Firecrawl.
    
    **When to use:**
    - Finding open source libraries
    - Exploring implementation examples
    - Researching project architecture
    - Discovering developer tools
    
    **Example queries:**
    - "Best TypeScript state management libraries"
    - "Redis caching implementations in Go"
    - "React component libraries for dashboards"
    
    **Features:**
    - Repository metadata (stars, forks, issues)
    - README extraction
    - License and dependency information
    - Recent activity and commit history
  </Accordion>
</AccordionGroup>

## Finance & Markets

<AccordionGroup>
  <Accordion title="Stocks" icon="chart-line">
    **Purpose**: Market data, charts, and financial analysis

## Choosing the Right Mode

**Let Scira decide**: In most cases, Scira will automatically select the best mode based on your query.

**Manual selection**: Choose a mode when you have specific requirements:

- **General research** → Web mode
- **Deep analysis** → Extreme mode
- **Academic citations** → Academic mode
- **Real-time trends** → X or Reddit mode
- **Financial data** → Stocks or Crypto mode
- **Media discovery** → YouTube or Spotify mode
- **Personal knowledge** → Connectors or Memory mode (Pro)
