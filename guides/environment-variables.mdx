Scira requires various API keys and configuration settings through environment variables. This guide provides a complete reference of all available environment variables.

<Info>
  Copy `.env.example` to `.env.local` (for development) or `.env` (for production) and configure the required variables.
</Info>

## AI API Keys

API keys for AI model providers. At minimum, you need OpenAI and Anthropic.

<ParamField path="XAI_API_KEY" type="string">
  xAI API key for Grok models (Grok 3, Grok 4, Grok Code)
  
  **Get your key**: [console.x.ai](https://console.x.ai)
</ParamField>

<ParamField path="OPENAI_API_KEY" type="string" required>
  OpenAI API key for GPT models
  
  **Get your key**: [platform.openai.com/api-keys](https://platform.openai.com/api-keys)
</ParamField>

<ParamField path="ANTHROPIC_API_KEY" type="string" required>
  Anthropic API key for Claude models (Haiku, Sonnet, Opus)
  
  **Get your key**: [console.anthropic.com](https://console.anthropic.com)
</ParamField>

<ParamField path="GROQ_API_KEY" type="string">
  Groq API key for fast inference
  
  **Get your key**: [console.groq.com](https://console.groq.com)
</ParamField>

<ParamField path="GOOGLE_GENERATIVE_AI_API_KEY" type="string">
  Google API key for Gemini models (Gemini 2.5 Flash, Gemini 2.5 Pro, Gemini 3)
  
  **Get your key**: [makersuite.google.com/app/apikey](https://makersuite.google.com/app/apikey)
</ParamField>

## Database & Storage

<ParamField path="DATABASE_URL" type="string" required>
  PostgreSQL database connection URL
  
  Format: `postgresql://user:password@host:port/database`
  
  **Example**: `postgresql://postgres:password@localhost:5432/scira`
  
  For production, use a managed PostgreSQL service like:
  - [Vercel Postgres](https://vercel.com/storage/postgres)
  - [Supabase](https://supabase.com)
  - [Neon](https://neon.tech)
  - [Railway](https://railway.app)
</ParamField>

<ParamField path="REDIS_URL" type="string" required>
  Redis connection URL for serverless caching and rate limiting
  
  Format: `redis://default:password@host:port`
  
  **Recommended**: [Upstash Redis](https://upstash.com/redis) for serverless Redis
</ParamField>

<ParamField path="BLOB_READ_WRITE_TOKEN" type="string">
  Vercel Blob storage token for file uploads
  
  Required for storing uploaded files (PDFs, images, etc.) and extreme search artifacts.
  
  **Get your token**: [vercel.com/storage/blob](https://vercel.com/storage/blob)
</ParamField>

## Authentication

<ParamField path="BETTER_AUTH_SECRET" type="string" required>
  Secret key for Better Auth session encryption
  
  Generate a secure random string:
  ```bash
  openssl rand -base64 32
  ```
  
  <Warning>
    Keep this secret secure. Never commit it to version control or expose it publicly.
  </Warning>
</ParamField>

### GitHub OAuth

<ParamField path="GITHUB_CLIENT_ID" type="string">
  GitHub OAuth application client ID
  
  **Create an OAuth app**: [github.com/settings/developers](https://github.com/settings/developers)
  
  Set callback URL to: `https://yourdomain.com/api/auth/callback/github`
</ParamField>

<ParamField path="GITHUB_CLIENT_SECRET" type="string">
  GitHub OAuth application client secret
</ParamField>

### Google OAuth

<ParamField path="GOOGLE_CLIENT_ID" type="string">

  **Create OAuth credentials**: [console.cloud.google.com/apis/credentials](https://console.cloud.google.com/apis/credentials)
  
  Set authorized redirect URI to: `https://yourdomain.com/api/auth/callback/google`
</ParamField>

<ParamField path="GOOGLE_CLIENT_SECRET" type="string">
  Google OAuth 2.0 client secret
</ParamField>

  Set callback URL to: `https://yourdomain.com/api/auth/callback/twitter`

## Client-side Environment Variables

These variables are exposed to the browser with the `NEXT_PUBLIC_` prefix.

<Warning>
  Only use `NEXT_PUBLIC_` prefix for non-sensitive data that can be safely exposed to clients.
</Warning>

<ParamField path="NEXT_PUBLIC_MAPBOX_TOKEN" type="string">

  **Get your token**: [mapbox.com/account/access-tokens](https://account.mapbox.com/access-tokens)

## Environment File Examples

### Development (.env.local)

```bash
# Minimal configuration for local development
OPENAI_API_KEY=sk-...
ANTHROPIC_API_KEY=sk-ant-...
EXA_API_KEY=...

DATABASE_URL=postgresql://postgres:password@localhost:5432/scira
REDIS_URL=redis://localhost:6379

BETTER_AUTH_SECRET=your-generated-secret-here

# Optional: Add more API keys as needed
```

### Production (.env)

```bash
# Production environment configuration
NODE_ENV=production

# Required AI providers
OPENAI_API_KEY=sk-...
ANTHROPIC_API_KEY=sk-ant-...
EXA_API_KEY=...

# Database (use managed PostgreSQL)
DATABASE_URL=postgresql://user:pass@host:5432/db?sslmode=require
REDIS_URL=rediss://default:pass@host:6379

# Authentication
BETTER_AUTH_SECRET=your-secure-production-secret

# OAuth providers (configure all you want to support)
GITHUB_CLIENT_ID=...
GITHUB_CLIENT_SECRET=...
GOOGLE_CLIENT_ID=...
GOOGLE_CLIENT_SECRET=...

# Storage
BLOB_READ_WRITE_TOKEN=...

# Add all other API keys for features you want to enable
```

## Security Best Practices

<Warning>
  **Never commit environment files to version control!**
  
  Ensure `.env`, `.env.local`, and `.env.production` are in your `.gitignore`.
</Warning>

## Validation

Scira validates environment variables at build time using `@t3-oss/env-nextjs`. Missing required variables will cause build failures with helpful error messages.
